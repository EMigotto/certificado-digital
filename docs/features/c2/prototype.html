C1. Inventário Centralizado de Certificados

Protótipos e design do MVP

Protótipos

  • 📋 Protótipo completo do mTLS Control Plane (MVP) Aprovado

    Protótipo interativo mostrando as 6 telas principais:
    1. Dashboard — heatmap de expiração e alertas (C3)
    2. Inventário centralizado — lista, busca, filtros (C1)
    3. Detalhe do certificado — metadados e ações (C1)
    4. Emitir certificado — formulário (C4)
    5. Audit Log — histórico de eventos (C5)
    6. API & CLI — documentação (integração)

Documentação Associada

  • 📄 Product Requirements Document (PRD)

    Especificação completa do recurso com requisitos funcionais, scope e suposições.

  • ✅ Acceptance Criteria

    Cenários em formato Gherkin (BDD) mapeados às telas do protótipo.

Destaques do Protótipo

  • Tela 02 — Inventário centralizado: Lista paginada de 2.847 certificados com busca por CN/SAN/serial/owner, filtros por expiração/ambiente/CA/owner, status visual (Crítico/Atenção/Válido) e navegação para detalhe.
  • Tela 03 — Detalhe do certificado: Exibição completa de metadados (CN, SANs, issuer, serial, datas, algoritmo, fingerprint SHA256), tags customizáveis, informações operacionais (owner, app, ambiente), e ações (renovar, download, revogar).
  • Requisitos não-funcionais: Suporta 10k+ certificados com paginação; filtro por "expira em < 30 dias" retorna em < 2 segundos.
